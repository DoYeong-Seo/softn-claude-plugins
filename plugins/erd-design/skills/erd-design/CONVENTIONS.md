# BlogN ERD 도메인 컨벤션

설계 검토·DDL 생성 시 **이 컨벤션을 우선 적용**한다. 일반적인 DBA 관점의 권고와 충돌하면 이 문서가 우선이다.

## 1. 논리명 표기 규칙 (용어사전 기반)

논리명은 **용어사전**으로 관리되며, 다음 마커는 **의도적**이다 — 오타로 보고하지 않는다.

| 마커 | 의미 | 예 |
|------|------|----|
| `[...]` | 업무영역(domain context) 묶음 표시 | `[게시판] 정보` 등 |
| 끝의 `]` | 해당 단어가 **약어**임을 표시 | `정보` = `INFORMATION`, `정보]` = `INFO` |

**검토 시 주의**:
- "논리명 끝에 `]` 가 붙어 있다"는 사실만으로 오타라고 판단하지 않는다.
- 사전 매핑(예: `정보] → INFO`)을 단순 텍스트 오타로 정리하지 않는다.
- 정상 케이스 예시: `실험실 게시판 정보]` (LABS_BOARD_INFO), `실험실 모델 정보]` (LABS_MODEL_INFO), `실험실 모델 배포 정보]` (LABS_MODEL_RELEASE_INFO), `투표 설명]`, `옵션 설명]`, `모델 설명]`.

## 2. 감사 컬럼 (BasicVO) 예외 사례

원칙적으로 모든 테이블은 BlogN BasicVO 표준 감사 컬럼 7개(`createDatetime`, `createUserId`, `createUserIp`, `modifyDatetime`, `modifyUserId`, `modifyUserIp`, `lockTimestamp`)를 가진다.

다만 **부모 엔티티의 화면에서 자식이 한꺼번에 입력·갱신되는 종속 테이블**은 부모의 감사 필드로 갈음하며, 자식 테이블에 감사 컬럼을 두지 않을 수 있다.

| 부모 | 자식(감사 컬럼 생략 가능) | 사유 |
|------|---------------------------|------|
| `LABS_POLL` | `LABS_POLL_OPTION` | 투표 생성/수정 화면에서 옵션이 함께 입력·갱신됨. 옵션 단독 변경 흐름 없음 |

**검토 시 주의**: 자식 테이블에 감사 컬럼이 없다는 사실만으로 BasicVO 표준 위반이라고 판정하지 않는다. 부모 화면 단위로 함께 입력되는 종속 테이블인지 먼저 확인한다.

## 3. i18n 키 컬럼 (`*_NAME_KEY` 패턴)

`*_NAME_KEY` 컬럼(예: `BOARD_NAME_KEY`, `MODEL_NAME_KEY`)은 **다국어(i18n) 메시지 키**로 사용된다 — 데이터 유일성을 보장하는 식별자가 아니다.

- 다국어 메시지 테이블의 `messageKey`로 매핑되어 표시 텍스트를 가져오는 용도다.
- 동일한 키가 여러 행에 존재할 수 있다 (예: 시스템 게시판 / 사용자 정의 게시판이 같은 키를 공유하는 경우).
- 따라서 `*_NAME_KEY` 컬럼에 **UNIQUE 제약을 강제하지 않는다.**

**검토 시 주의**: "키"라는 컬럼명만으로 UNIQUE 누락을 지적하지 않는다. 비즈니스 의미가 식별자인지 i18n 키인지부터 확인한다.
